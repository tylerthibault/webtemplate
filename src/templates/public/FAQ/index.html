üöÄ Getting Started

Setting up the template is straightforward:

  1. Clone the repository to your local machine
  2. Create a virtual environment: python -m venv .venv
  3. Activate the virtual environment
  4. Install dependencies: pip install -r requirements.txt
  5. Run the application: python run.py

The application will be available at http://localhost:5000

  • Python: 3.8 or higher
  • Operating System: Windows, macOS, or Linux
  • Memory: 512MB RAM minimum
  • Storage: 100MB free space
  • Browser: Modern browser with JavaScript enabled

üîß Technical Questions

Our MVC implementation follows strict separation of concerns:

  • Models: Located in src/models/ - Handle database schema and data relationships
  • Views: Located in src/templates/ - Jinja2 templates with inheritance structure
  • Controllers: Located in src/controllers/ - Thin routing layer that delegates to logic layer
  • Logic Layer: Located in src/logic/ - Business rules and application logic

The template uses SQLAlchemy ORM, which supports multiple database systems:

  • SQLite: Default for development (no setup required)
  • PostgreSQL: Recommended for production
  • MySQL: Fully supported
  • SQL Server: Supported with appropriate drivers

Change the DATABASE_URL in your .env file to switch databases.

Follow these steps to add new features:

  1. Create a route: Add to appropriate controller in src/controllers/
  2. Add business logic: Create service functions in src/logic/
  3. Create templates: Add HTML templates in src/templates/
  4. Add models: If needed, create database models in src/models/
  5. Write tests: Add tests for your new functionality

üîê Security Questions

Our custom authentication system includes:

  • Password Hashing: Bcrypt with cost factor 12
  • Session Management: "Coat hanger" pattern with database-stored sessions
  • Session Timeout: 10 minutes of inactivity
  • CSRF Protection: Built-in token validation
  • Login Required Decorator: Easy route protection

The template includes comprehensive security features:

  • CSRF Protection: All forms include CSRF tokens
  • XSS Prevention: Jinja2 auto-escaping enabled
  • SQL Injection Protection: SQLAlchemy parameterized queries
  • Secure Sessions: HTTPOnly and secure cookie flags
  • Input Validation: Server-side validation on all inputs
  • Password Security: Strong hashing and complexity requirements

For production deployment, ensure you:

  1. Set a strong SECRET_KEY in environment variables
  2. Use a production database (PostgreSQL recommended)
  3. Enable HTTPS/TLS encryption
  4. Set appropriate CORS policies
  5. Configure proper logging and monitoring
  6. Use environment variables for all sensitive configuration
  7. Consider using a reverse proxy (nginx, Apache)
